# 「これどう捨てる？」データ出典・照合 説明資料

開発元: ほほ笑みラボ（飯田市）／ rabo.hohoemi@gmail.com
アプリ版: ベータ版（データ version 1.2.0 時点）

---

## 0. 大前提（最初に伝えたいこと）

本アプリは **分別ルールを自分で考えたり推測したりしません**。

```
ごみを撮影 → AI は「品目名」を当てるだけ
            → その品目名で、アプリ内に持っている
              「飯田市公式データ由来の辞書」を照合して、ルールを表示
```

- **AI（Gemini）の役割は「品目名の特定」だけ**。「燃やすごみ」「第何曜日」などの分別ルールは **公式データに基づくアプリ内辞書から表示**します（＝AI にルールを言わせない「ハルシネーション・ゼロ」設計）。
- つまり、表示される分別ルールの正しさは **元データの正しさ＝飯田市公式データの正しさ** に一致します。だからこそ「どの公式データを使っているか」を正直にお見せします。

## 2. さんあ〜る品目データの扱い（詳しく）

担当者が一番気にされる点なので、正直に整理します。

- **何を取ったか**: 公式「さんあ〜る」に掲載の **589品目**（品目名・分別区分・出し方の説明・検索キーワード）。
- **どう変換したか**:
  - 品目名 → アプリの検索辞書の見出しに
  - 分別区分 → アプリの11カテゴリにマッピング（複数区分の品目は、主区分をカテゴリにし、部品ごとの捨て方は説明文に保持）
  - 検索キーワード（`data-keyword`）→ 別名として保持（検索ヒット率の維持）
- **画像・ロゴ・キャラクター等は使用していません**（テキストの分別情報のみ）。
- **表示**: 各画面に「**ベータ版**」「**最新・正確な情報は飯田市公式へ**」を明記し、**市公式アプリではない**と分かる作りにしています。

### 正直にお伝えする差異・未確定点（リリース前に市と確認したい）

- **さんあ〜る（品目）と 計画表PDF（収集）で分類体系が一部異なります**（特にプラスチックの分け方）。本アプリは「品目の分別＝さんあ〜る」「収集日＝PDF」を正本にしています。ここの整合は確認したい点です。
- **ガイドブックとカレンダーで食い違う箇所**（例: 稲葉クリーンセンター料金 220円/180円、家電引取業者の住所・電話）があり、どちらが現行か **市に確認したい**。
- **データの利用許諾は未取得**です。**まさにこの点をご相談したい**（→ §4）。

## 3. データの流れ（プライバシー設計とあわせて）

```
[公式データ]                          [アプリ（端末内）]
 さんあ〜る589品目 ──変換──▶ items.json（バンドル）
 計画表PDF ───────変換──▶ patterns/areas/facilities…（バンドル）
                                         │
[カメラ判定の流れ]                        │ 品目名で照合
 撮影画像 ─▶ 自社Worker ─▶ Gemini ─▶「品目名」だけ返す ─▶ 辞書照合 ─▶ ルール表示
              （Cloudflare）   （Google）
              ▲ 画像はここで判定後すぐ破棄・保存もログも残さない
```

- **撮影画像はサーバーに保存しません**（判定後即破棄・ログにも残さない）。**ログイン不要・個人情報の取得は最小**。
- 分別ルールの表示は **すべて端末内の辞書（公式データ由来）** から。通信が無くてもカメラ判定以外は動きます。

## 4. ご相談したいこと（さんあ〜る担当者さまへ）

1. **公式分別データ（さんあ〜る）の利用・引用許諾の整理**
   - 現在 589品目をもとに辞書を作成しています。**どの範囲なら市民向け無料アプリで使ってよいか**を確認させてください。
   - 表示方法・出典明記・「市公式ではない」明示など、ご要望に合わせます。**「使わないで」と言われた範囲は使いません**。
2. **分別区分の確認（気になる品目だけでも）**
   - 例: 「時計＝埋立ごみ」で合っているか等。
3. **（将来）全市分データのご提供・共同メンテ**
   - データ差し替えだけで更新でき、**アプリ再申請なしで反映できる仕組み（OTA）は実装済み**です。

### お返しできること（双方向）

- アプリには「**未収録品目の報告**」機能を実装済み。市民が調べたのに **辞書（＝公式データ）に無かった品目** が運用側に届きます。集計すれば「**市民が実際に困っている、公式データの穴リスト**」になり、**市のデータ整備・問い合わせ削減に還元**できます（送るのは品目名テキストのみ・画像/位置情報/個人情報なし）。
