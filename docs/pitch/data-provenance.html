0. 大前提

本アプリは分別ルールを自分で考えたり推測したりしません。

ごみを撮影 → AI は「品目名」を当てるだけ
            → その品目名で、アプリ内に持っている
              「飯田市公式データ由来の辞書」を照合して、ルールを表示

AI（Gemini）の役割は「品目名の特定」だけ。分別ルールは公式データに基づくアプリ内辞書から表示します（＝AIにルールを言わせない「ハルシネーション・ゼロ」設計）。表示されるルールの正しさは元データ＝飯田市公式データの正しさに一致します。

2. さんあ〜る品目データの扱い（詳しく）

  • 何を取ったか: 公式「さんあ〜る」掲載の589品目（品目名・分別区分・出し方の説明・検索キーワード）。
  • どう変換したか: 品目名→検索見出し／分別区分→11カテゴリにマッピング（複数区分は主区分をカテゴリにし、部品ごとの捨て方は説明文に保持）／検索キーワード→別名として保持。
  • 画像・ロゴ・キャラクター等は使用していません（テキストの分別情報のみ）。
  • 表示: 各画面に「ベータ版」「最新・正確な情報は飯田市公式へ」を明記し、市公式アプリではないと分かる作り。

正直にお伝えする差異・未確定点（リリース前に市と確認したい）

  • さんあ〜る（品目）と 計画表PDF（収集）で分類体系が一部異なります（特にプラスチックの分け方）。本アプリは「品目＝さんあ〜る」「収集日＝PDF」を正本にしています。
  • ガイドブックとカレンダーで食い違う箇所（例: 稲葉クリーンセンター料金 220円/180円、家電引取業者の住所・電話）があり、どちらが現行か市に確認したい。
  • データの利用許諾は未取得です。まさにこの点をご相談したい（→ 別紙 許諾依頼）。

3. データの流れ（プライバシー設計とあわせて）

[公式データ]                          [アプリ（端末内）]
 さんあ〜る589品目 ──変換──▶ items.json（バンドル）
 計画表PDF ───────変換──▶ patterns/areas/facilities…（バンドル）
                                         │
[カメラ判定の流れ]                        │ 品目名で照合
 撮影画像 ─▶ 自社Worker ─▶ Gemini ─▶「品目名」だけ返す ─▶ 辞書照合 ─▶ ルール表示
              （Cloudflare）   （Google）
              ▲ 画像はここで判定後すぐ破棄・保存もログも残さない

撮影画像はサーバーに保存しません（判定後即破棄・ログにも残さない）。ログイン不要・個人情報の取得は最小。分別ルールの表示はすべて端末内辞書（公式データ由来）から。

4. ご相談したいこと

  1. 公式分別データ（さんあ〜る）の利用・引用許諾の整理 — 市民向け無料アプリで使ってよい範囲を確認したい。表示方法・出典明記・「市公式ではない」明示はご要望に合わせます。「使わないで」と言われた範囲は使いません。
  2. 分別区分の確認（気になる品目だけでも。例:「時計＝埋立ごみ」で合っているか）。
  3. （将来）全市分データのご提供・共同メンテ — データ差し替えだけで更新でき、アプリ再申請なしで反映（OTA）を実装済み。

お返しできること: 「未収録品目の報告」機能で、市民が調べたのに辞書（＝公式データ）に無かった品目が運用側に届きます。集計すれば「市民が実際に困っている、公式データの穴リスト」になり、市のデータ整備・問い合わせ削減に還元できます（送るのは品目名テキストのみ・画像/位置情報/個人情報なし）。
