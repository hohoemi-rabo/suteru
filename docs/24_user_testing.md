# 24. ユーザーテスト（ほほ笑みラボ生徒）

> 関連: `REQUIREMENTS.md` §1.3, §9 フェーズ1
> ステータス: 未着手

## 目的

ほほ笑みラボの生徒さん約20名にプレビュー版を配布し、実利用フィードバックを集める。**シニア層が実際に使えるかの最終チェック** がMVPの肝。

## 完了条件

- 5名以上の生徒さんが実機で使ってフィードバックを返している
- 重大な使いにくさが特定されて修正済み
- 行政アピール時の「実利用者の声」として使える証言が集まっている

## 注意点

- **テスター負担を最小に**：使い方を全部教えすぎず、自然に触らせる
- 「使えなかった」「わからなかった」を聞き出すには信頼関係が必要。ほほ笑みラボの場で実施するのが理想
- 集めた個人情報（端末ログ、画像）は **絶対に持ち帰らない**（プライバシー設計の根幹）
- フィードバックは紙やボイスメモで取り、後でデジタル化。テスト中はテスター観察に集中
