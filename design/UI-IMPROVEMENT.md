# UI改善 実装指示書

> 「これどう捨てる？」のUIを「明るく・親しみやすく・ポップで生活感のある」トーンに整える指示書。
> Claude Codeはこれと `design/Colors.ts` を読んで、全画面を統一ルールで改善する。

## 0. 前提：デザイントークンの導入

まず `design/Colors.ts` の内容を `constants/Colors.ts` に反映（既存のColors.tsとマージ）。
**以降、色・余白・角丸・文字サイズは必ずこのトークン経由で参照する。ハードコード禁止。**

## 1. 全画面共通のルール

### 配色

- 画面背景: `Palette.bg.page`（薄い青 #EEF4FB）
- カード・パネル: `Palette.bg.surface`（白）
- 主役の緑: `Palette.green[400]`（#1D9E75）— カメラボタン、アクティブなタブ、主要ボタン
- 地区・リンク・情報: `Palette.blue`系
- 見出しの濃い文字: `Palette.green[900]` か `Palette.text.primary`

### 余白・角丸

- カードの角丸: `Radius.lg`（16）
- カード内の余白: 上下 `Spacing.md`〜`lg`、左右 `Spacing.lg`
- カード間の間隔: `Spacing.md`（12）以上空ける（今は詰まっているので広げる）
- 検索バー・バッジ・主要ボタン: `Radius.pill`（999）で丸く

### 文字

- 画面タイトル: `FontSize.title`(21) / `FontWeight.medium`
- 本文: `FontSize.body`(15)
- 補足・注意書き: `FontSize.small`(13) / `Palette.text.secondary`
- **文字数を減らす**: 注意書きは1〜2行に圧縮。長い説明はたたむ

### ベータバッジ（共通コンポーネント化）

`components/BetaBadge.tsx` を作る:
- 背景 `Palette.green[200]`、文字 `Palette.green[600]`、`FontSize.caption`、`Radius.pill`、padding 3px 9px
- 各画面タイトルの右隣に置く

## 2. ホーム画面（app/(tabs)/index.tsx）

参照: 提示済みのモックアップのトーン

### 変更点

1. **カメラボタンの説明文を円の外に出す**
   - 今: 「ごみを撮るだけで分別が分かる」が円内に入って切れている
   - 改善: 円の中は「アイコン＋写真でしらべる」だけ。説明文は円の **下** に `Palette.green[600]` で配置
2. **カメラボタンのサイズ**: 直径180〜190px程度。色は `Palette.green[400]`、影なし、フラット
3. **検索バーをピル型に**: `Radius.pill`、白背景、薄いボーダー
4. **地区カードは青系**: 背景 `Palette.blue[50]`、アイコンと「変更」は `Palette.blue[600]`、地区名は `Palette.blue[900]`
5. **ベータ注意書きを圧縮**: アイコン＋1〜2行。「公式サイトで確認 ›」をリンクに
6. **診断パネル**: `__DEV__` でのみ表示（本番ビルドで出さない）。これは既に実装済みかもしれないが確認

## 3. 収集日画面（app/(tabs)/schedule.tsx）

今のスクショは情報量が多く、テキストが詰まっている印象。整理する。

### 変更点

1. **地区サマリー行の圧縮**
   - 今: 「燃やすごみ火金 / プラ資源木 / 埋立特定第2・4水 / 金属紙第1・3木」が2行でぎっしり
   - 改善: 各カテゴリを **色ドット＋短い曜日** のチップにして横並び。または折りたたんで「収集スケジュールを見る」で展開
2. **「次の収集」カード**: 今の緑カードは良い。`Palette.green[100]`背景、日付は `FontSize.hero`(28) で大きく。「今日」バッジを強調
3. **リスト/カレンダー切替トグル**: アクティブ側を `Palette.green[400]` の文字＋白背景で明示（今の実装でOK、色だけトークンに合わせる）
4. **カレンダーの曜日ヘッダー**: 土=青、日=赤は今のままでOK
5. **凡例**: カード化して `Palette.bg.surface`、色ドットとラベルを2列グリッドで整列
6. **カテゴリ別リスト**: 各行に色ドット。「明後日」などの相対表現は良いので残す
7. **「プラスチッ...」の省略**: 列幅が足りず品目名が切れている。カテゴリ名を `CategoryColors` の `label` で表示し、省略せず2行許容 or 列幅調整

## 4. 施設画面（app/(tabs)/facilities.tsx）

### 変更点

1. **リサイクルステーションカード**: 青系アイコン背景は良い。`Palette.blue[50]` 背景でカード化
2. **持ち込み施設カード**: 各施設を白カードに。施設名は `FontSize.subtitle`/`medium`
3. **電話ボタン**: 今の緑のボタンは良いが、`Palette.green[400]` に統一。`Radius.pill`、電話アイコン＋番号
4. **情報行のアイコン**: 住所📍・日時📅・時計🕐・料金などのアイコンは `Palette.text.secondary`、行間を `Spacing.sm` 空ける

6. **セクション見出し**（「持ち込み施設」「資源ごみ受入業者」）: `FontSize.title` より少し小さめ、`Palette.green[900]`

2. **「※上記以外のエリアは近日対応予定」**: 今は赤背景。`Palette.danger.bg` のままでもいいが、ネガティブすぎるなら `Palette.blue[50]`＋`Palette.blue[600]`文字の「お知らせ」トーンに変更を検討
3. **通知トグル**: 今のカードは良い。説明文は `FontSize.small`
4. **通知時刻チップ**: 選択中を `Palette.green[400]` 文字＋`Palette.green[50]`背景＋緑ボーダー（今の実装でOK）
5. **データセクション**: バージョン表示と「データ更新を確認」ボタン。ボタンは `Palette.green[400]`、`Radius.pill`

- アクティブタブ: `Palette.green[400]`（アイコン＋ラベル）
- 非アクティブ: `Palette.text.tertiary`

- アイコンは今のまま（home / calendar / map-pin / settings 相当）

## 7. 実装の順序（推奨）

1. `constants/Colors.ts` にトークンを反映
2. `components/BetaBadge.tsx` など共通コンポーネントを作る
3. タブバーの色をトークンに統一（全画面に影響）
4. ホーム → 収集日 → 施設 → 設定 の順に各画面を適用
5. 実機で確認しながら余白・文字サイズを微調整

## 8. やらないこと（スコープ外）

- 機能の追加・変更（今回はあくまで見た目の改善）
- 新しいライブラリの導入（既存のスタイリング手法で実現）
- ダークモード対応（フェーズ2以降）
- アニメーションの追加（最小限のまま）

## 補足：トーンの指針

「明るく・親しみやすく・ポップで生活感のある」とは:
- **明るく**: 彩度高めの緑を主役に、白とのコントラストをはっきり
- **親しみやすく**: 角を丸める（ピル型ボタン、丸いカメラボタン）
- **ポップ**: 色ドットでカテゴリを楽しく区別、余白で軽やかに
- **生活感**: 専門用語より日常語（「写真でしらべる」「明日のごみ出し」）

ただし行政アピールも兼ねるので、**ポップにしすぎて軽薄に見えないバランス** を保つ。情報の正確さ・読みやすさが最優先。
